### 4. `markWord_with_hash` 的 shift=8 与本地源码一致

```cpp
// my_jvm/src/oops/markOop.hpp
inline markOop markWord_with_hash(markOop m, uintptr_t hash) {
    uintptr_t hash_mask_in_place = ((uintptr_t)0x7FFFFFFF) << 8;
    uintptr_t v = m->raw_value();
    v = (v & ~hash_mask_in_place) | ((hash & 0x7FFFFFFF) << 8);
    return (markOop)v;
}
```

**本地源码验证**（`markOop.hpp`）：
```cpp
markOop copy_set_hash(intptr_t hash) const {
    intptr_t tmp = value() & (~hash_mask_in_place);
    tmp |= ((hash & hash_mask) << hash_shift);  // hash_shift = 8
    return (markOop)tmp;
}
```
✅ 完全一致。

### 6. `is_locked()` / `is_unlocked()` 的判断逻辑与本地源码一致

```cpp
// my_jvm/src/oops/markOop.hpp
bool is_locked() const {
    return (value() & 0x3) != unlocked_value;  // != 1
}
bool is_unlocked() const {
    return (value() & 0x7) == unlocked_value;  // == 1
}
```

**本地源码验证**（`markOop.hpp`）：
```cpp
bool is_locked()   const {
    return (mask_bits(value(), lock_mask_in_place) != unlocked_value);
}
bool is_unlocked() const {
    return (mask_bits(value(), biased_lock_mask_in_place) == unlocked_value);
}
```

注意：`is_unlocked()` 用的是 `biased_lock_mask_in_place`（低3位），而不是 `lock_mask_in_place`（低2位）。这是因为偏向锁的低3位是 `101`，不等于 `unlocked_value=1`，所以用低3位判断可以同时排除偏向锁状态。你的实现用 `0x7`（低3位）是正确的。✅

## ❌ 仍然存在的问题（严格基于本地源码）

### 问题 1：`markWord_biased` 里 `thread << 10` 是错的（逻辑错误）

**你的代码**（`markOop.hpp` 第 97 行）：
```cpp
inline markOop markWord_biased(void* thread, int age, int epoch) {
    uintptr_t ptr = ((uintptr_t)thread << 10) |   // ← 左移 10 位，错误！
                    ((uintptr_t)epoch << 8) |
                    ((uintptr_t)age << 3) |
                    0x5;
    return (markOop)ptr;
}
```

**本地源码**（`markOop.hpp` 第 310-317 行）：
```cpp
static markOop encode(JavaThread* thread, uint age, int bias_epoch) {
    intptr_t tmp = (intptr_t) thread;   // ← thread 指针直接用，不左移！
    assert(UseBiasedLocking && ((tmp & (epoch_mask_in_place | age_mask_in_place | biased_lock_mask_in_place)) == 0),
           "misaligned JavaThread pointer");
    return (markOop) (tmp | (bias_epoch << epoch_shift) | (age << age_shift) | biased_lock_pattern);
    //                       epoch_shift = hash_shift = 8
    //                                                   age_shift = 3
}
```

本地源码的注释明确说明了原因（`markOop.hpp` 第 80-82 行）：
```
// The runtime system aligns all JavaThread* pointers to
// a very large value (currently 128 bytes (32bVM) or 256 bytes (64bVM))
// to make room for the age bits & the epoch bits
```

`JavaThread*` 指针本身就是 256 字节对齐的（64位下），低 8 位天然为 0，所以直接 `OR` 进去就行，**不需要左移**。你写的 `thread << 10` 会把线程指针的高位截断，是错的。

**正确写法**：
```cpp
inline markOop markWord_biased(void* thread, int age, int epoch) {
    uintptr_t ptr = (uintptr_t)thread;   // 不需要左移！
    ptr |= ((uintptr_t)epoch << 8) |     // epoch_shift = hash_shift = 8
           ((uintptr_t)age   << 3) |     // age_shift = 3
           0x5;                          // biased_lock_pattern = 5
    return (markOop)ptr;
}
```
