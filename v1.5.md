## ✅ 本版新增的正确点

### 1. `oopDesc::_mark` 加上了 `volatile` ✅

```cpp
// my_jvm/src/oops/oop.hpp
volatile markOop _mark;
```

**本地源码验证**（`oop.hpp` 第 54 行）：
```cpp

```
完全一致。这是并发安全的关键，多线程并发修改对象头（锁膨胀、偏向锁撤销、GC 标记）时必须有 `volatile`。

### 2. `markOop.hpp` 的 `markWord_biased` 注释非常准确 ✅

```cpp
// JavaThread 指针是 256 字节对齐的，低8位天然为0，所以直接 OR 进去即可，不需要左移
// age_shift = 3, epoch_shift = 8
inline markOop markWord_biased(void* thread, int age, int epoch) {
    uintptr_t ptr = (uintptr_t)thread;  // 直接用，不需要左移！
```

注释准确解释了为什么不需要左移，说明你真正理解了这个设计。

### 3. `metadata.hpp` 的继承体系设计合理 ✅

```
MetaspaceObj → Metadata → Klass / Method
```

与本地源码一致（本地源码：`MetaspaceObj → Metadata → Klass / Method`）。

特别说明：本地源码的 `is_instance_klass()` 是**非虚函数**，通过 `_layout_helper` 的正负号判断：

**你的代码**（`instanceKlass.hpp` 第 14-21 行）：

enum InstanceClassState {

### 问题 3：`_init_state` 的类型应为 `u1`（连续两版未修复）

**你的代码**（`instanceKlass.hpp` 第 88 行）：
```cpp
InstanceClassState _init_state;   // ← 4字节枚举
```

**本地源码**（`instanceKlass.hpp` 第 248 行）：
```cpp
u1 _init_state;   // ← 1字节！
```

本地源码注释说明了原因：
```
// Place the _init_state here to utilize the unused 2-byte after
// _idnum_allocated_count.
```

用 `u1` 是为了节省内存，利用结构体中的空隙字节。你用 `InstanceClassState`（4字节枚举）会多占 3 字节。

### 问题 5：`Method` 缺少 `ConstMethod*` 字段（最重要的字段）

**你的代码**（`method.hpp`）：
```cpp
class Method : public Metadata {
private:
    u2 _access_flags;
    u2 _name_index;        // ← 这些字段在本地源码里都在 ConstMethod 里！
    u2 _signature_index;   // ← 同上
    u4 _code_size;         // ← 同上
    u2 _method_parameters_length;
    u2 _max_locals;        // ← 同上
    u2 _max_stack;         // ← 同上
    u2 _exception_table_length;
};
```

**本地源码**（`method.hpp` 第 68-80 行）中，`Method` 的核心字段：
```cpp
class Method : public Metadata {
    ConstMethod*      _constMethod;    // ← 最重要的字段！字节码、异常表等都在这里
    MethodData*       _method_data;    // profiling 数据
    MethodCounters*   _method_counters; // 调用计数器
    AccessFlags       _access_flags;
    int               _vtable_index;   // vtable 索引（多种编码）
    u2                _intrinsic_id;
    mutable u2        _flags;
    address           _i2i_entry;      // 解释器入口
    volatile address  _from_compiled_entry;
    CompiledMethod* volatile _code;    // 编译后的代码
    volatile address  _from_interpreted_entry;
};
```

本地源码里 `_name_index`、`_signature_index`、`_code_size`、`_max_stack`、`_max_locals` 等字段都在 `ConstMethod` 里：
```cpp
// constMethod.hpp
class ConstMethod : public MetaspaceObj {
    uint64_t          _fingerprint;
    ConstantPool*     _constants;      // 常量池
    Array<u1>*        _stackmap_data;
    int               _constMethod_size;
    u2                _flags;
    u2                _code_size;      // ← 字节码大小在 ConstMethod 里
    u2                _name_index;     // ← 方法名索引在 ConstMethod 里
    u2                _signature_index;// ← 签名索引在 ConstMethod 里
    u2                _method_idnum;
    u2                _max_stack;      // ← max_stack 在 ConstMethod 里
    u2                _max_locals;     // ← max_locals 在 ConstMethod 里
    u2                _size_of_parameters;
};
```

你的 `Method` 设计把 `ConstMethod` 的字段直接内联进来了，这是一种简化，但要知道本地源码里 `Method` 和 `ConstMethod` 是分开的两个结构，原因是：
- `ConstMethod` 存放**只读数据**（字节码、常量池引用、名称等），一旦加载就不变
- `Method` 存放**运行时数据**（编译代码指针、计数器、vtable 索引等），会频繁变化
- 分开存放有利于 JIT 编译器的缓存优化
